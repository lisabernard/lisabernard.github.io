Programming in Python

In December 2016, I finished a Python programming course. During that time, I explored the use of basic programming concepts, such as: if statements, functions, arrays, file input/output and loops. By the end of the course, I had programmed a variety of console-based, graphical and hardware interface applications. Below, I have an embedded battleship application that I programmed as part of my course.

Before I dove into developing my Battleship application, I didn’t know how to use a 2d array. So as I worked on it, I learned how put file input into a 2D array, update the graphics, end the game if the user sinks all the ships or runs out of ammo and tell the user when they’ve sunk a ship (that part was for bonus points).

If I were to redo this program, I would simplify the code and add in more comments to make it easier to follow. I’d also add more features, such as high scores and a user specified ammo count.

PCB/Circuit Board Design

In addition to learning 3D CAD, I took a PCB (Printed Circuit Board) design course. During which I was taught how to make schematics, custom footprints and pad stacks as well as layout and route PCBs.

My final project as was a calculator Arduino shield. This was the first assignment that I had to take a schematic created in a different program (KiCad) and redo it in Orcad. So, during the process of creating this shield, I learned about schematical differences between OrCad and KiCad, making custom footprints and padstacks (I only used custom ones to create this board).

If I were to redo this assignment, I would probably try to use less vias (more as a personal challenge than anything). In the future I would love to make more PCB boards and possibly learn how to use KiCad since it's a free open source program.
